# Screen Studio 완벽 모작을 목표로 삼는다

> Supersedes [0003](./0003-export-gif-only-for-dooray.md) (export GIF 단일), 폐기: "경량 편집(Light Edit)" 원칙.

Recap은 그동안 "Dooray 첨부용 짧은 데모 GIF"라는 목적 아래 기능을 좁혀왔다(GIF 단일 export, 경량 편집만 허용). 그러나 명확한 품질 레퍼런스 없이 목적을 좁혀가며 개발한 결과, 산출물 품질이 기대보다 떨어졌다.

리서치 결과 X(트위터)에 올라오는 세련된 개발자 데모 영상의 사실상 표준은 **Screen Studio**(macOS)이며, 세련됨의 실체는 자동 후편집(커서 추종 자동 줌, 커서 스무딩, 배경 패딩)과 고품질 렌더(최대 4K 60fps MP4)다. 또한 그 영상들의 실제 산출 포맷은 GIF가 아니라 **루핑 MP4(H.264)**다 — X는 GIF 업로드조차 서버에서 MP4로 트랜스코딩한다.

그래서 **Screen Studio의 UI·유저 플로우·기능·품질을 그대로 모작하는 것을 goal로 확정**한다. 모작을 완성한 뒤에 개선·차별화를 논한다. 구체적 레퍼런스를 기준점으로 삼아 "목적 없는 개발로 인한 품질 저하"를 막는 것이 핵심 근거다.

## Considered Options

- **Dooray GIF 도구 유지(현행)** — 목적은 명확하나 품질 기준이 없어 산출물이 기대 이하였다. 탈락.
- **UI 패턴만 복제, 능력은 경량 편집 유지 (단계적)** — 비용은 작지만 "무엇이 완성인가"의 기준이 다시 모호해진다. 레퍼런스를 부분 채택하면 판단이 매번 재발생한다. 탈락.
- **Screen Studio 완벽 모작 (채택)** — 컷/분할·속도 조절 등 편집 능력까지 포함해 그대로 복제. 기능 하나하나에 "Screen Studio는 어떻게 하는가"라는 판정 기준이 생긴다.

## Consequences

- export는 MP4·GIF 듀얼 출력으로 복원된다(0003 폐기). 품질 상한은 Screen Studio 동등 이상: MP4 최대 4K 60fps, 품질 슬라이더. ADR 0003에서 제거한 MP4 인코딩 경로(`mediabunny` 등)를 되살리거나 재구현해야 한다.
- "경량 편집" 원칙(트림+줌+배경만, 컷·속도 조절 금지)을 폐기한다. 타임라인은 컷/분할·속도 조절을 수용해야 하므로 "단일 구간 + 렌더 레시피" 구조에서 클립 시퀀스 구조로의 변경이 필요하다.
- 기능 판정 기준이 단순해진다: 의문이 생기면 Screen Studio의 실제 동작을 관찰하고 그대로 따른다.
- **시각적 충실도는 픽셀 근접 모작으로 판정한다**: 배경 프리셋·그림자·줌 이징·모션 블러·커서 스무딩 등 미적/모션 파라미터를 Screen Studio 기본값에 수치까지 맞추고, 같은 화면을 두 앱으로 녹화해 나란히 비교하는 것으로 "똑같다"를 확인한다. "느낌 동급"이 아니라 관찰된 레퍼런스 일치가 기준이다.
- Dooray 첨부는 여러 목적지 중 하나로 격하된다(GIF export가 이를 담당).
- **모작 대상에 Recap 앱 자체의 UI 크롬도 포함한다** — 레이아웃·동선뿐 아니라 색·타이포·아이콘·간격까지 Screen Studio에 맞춘다. 이에 따라 모노크롬 디자인 시스템(#115)은 보류한다. 모작을 완성한 뒤 별도의 새 디자인을 입히는 것을 후속 단계로 둔다.
- Screen Studio 스펙·UX 관찰 기록은 리서치 문서로 유지한다(docs/research/).
- **모작 범위는 무음 화면 데모 코어로 한정한다**: 녹화(전체화면/윈도우/영역)→에디터(자동/수동 줌, 커서, 배경, 트림, 컷/분할, 속도 조절)→export(MP4/GIF). 오디오(마이크·시스템 오디오·클릭 사운드), 웹캠 PIP, 자동 자막, iPhone 미러링은 모작 대상에서 제외한다. 단 클릭 위치 기반 자동 줌인/줌아웃과 클릭 시각 효과는 포함한다(Screen Studio와 동일하게 자동 줌 트리거는 클릭뿐이며, 키입력 트리거 줌은 없다).
- **격차를 메우는 실행 순서**: ② MP4 출력 복원 → ④ 모션 품질 픽셀 근접(줌 이징·모션 블러·hold push-in) → ① 앱 크롬 픽셀 모작(온보딩·녹화바·에디터) → ③ 컷/분할·속도 조절(데이터 모델을 단일 레시피에서 클립 시퀀스로 전환). MP4를 먼저 두는 이유는 픽셀 비교를 GIF(256색)가 아닌 원화질로 해야 정확하고 ④의 전제가 되기 때문이며, 데이터 모델 수술이 필요한 ③을 코어 안정 후 마지막에 둔다. 시각 레퍼런스는 `docs/research/screen-studio-visual-spec.md` + `screen-studio-samples/`.
