# Screen Studio 시각/UX 모작 스펙 (초안)

ADR 0004(Screen Studio 완벽 모작)를 위한 시각·모션 레퍼런스. 공개 이미지·공식 가이드에서 관찰 가능한 것을 먼저 채우고, 실물 확인이 필요한 항목은 "사용자 자료 필요"로 분리했다. 태그: `[관찰됨]` = 실제 이미지/공식 문서 확인, `[추정]` = 육안/단일 출처 추정.

## 사용 과정 서사 (screen.studio 랜딩 전체 스크롤 — 공식 카피)

Screen Studio가 스스로 설명하는 워크플로 순서와 핵심 동작(공식 문구 근거):

1. **자동 줌**: 커서/액션에 자동 줌인. 소셜용 export 시 줌을 자동 재조정. 수동 줌도 타임라인에서 드래그로 추가하면 애니메이션 자동 생성.
2. **기본 자동 애니메이션**: 커서 크기 확대(녹화 후에도 변경 가능), 흔들림→부드러운 glide, 커서 자동 숨김(애니메이션), **루프용 커서 초기위치 복귀**, 저해상도 시스템 커서를 고해상도로 교체.
3. **스타일/브랜딩**: 배경·외곽 여백·그림자·inset 변경. **원클릭 출력 종횡비 변경 → 모든 애니메이션 자동 재조정**.
4. **편집**: "Easily trim, cut, or speed up" — 트림·컷·속도 조절이 공식 기능.
5. **Export & Share**: 목적지(web/social/추가편집)를 정하면 최적 설정 자동 선택. **up to 4K 60fps 또는 최적화 GIF**. 클립보드 복사 후 아무 앱에 붙여넣기.
6. (우리 범위 밖) 웹캠 셀피 오버레이(커서 안 가리게 살짝 줌아웃), 음성 정규화+노이즈 제거, 로컬 자막, iPhone/iPad USB 녹화+기기 프레임 자동감지, 키보드 단축키 표시.

우리 범위(무음 코어)와의 대비: 4·5는 정면 목표. 2의 "커서 크기 후편집 변경·자동 숨김·루프 복귀"와 3의 "종횡비 원클릭+애니메이션 재조정"은 현행 Recap에 없거나 부분적이라 격차 항목이다.

## 모션 특성 (모작의 핵심)

- **화면 애니메이션 모드 2종**: Focused(빠른 안정화, 텍스트용) vs Smooth(유려, 크리에이티브용). `[관찰됨 — 공식]`
- **커서 속도 프리셋**: Smooth / Medium / Rapid / None. `[관찰됨 — 공식]`
- **모션 블러**: 3영역 개별 슬라이더(커서 이동 / 줌인·아웃 / 패닝). cinematic 목적 명시. `[관찰됨 — 공식]`
- **Instant Zoom**: 애니메이션 없이 즉시 확대하는 옵션 별도 존재(Instant Animation 토글). `[관찰됨 — 공식]`
- **스프링 물리 가능성**: 제3자(daveswift.com)가 줌 애니메이션에 Tension/Friction/Mass 파라미터(Slow/Mellow/Quick/Rapid 프리셋 + Customize)를 언급. 사실이면 이징 곡선이 아니라 **스프링(질량-감쇠-장력) 모델** → cubic-bezier 근사보다 spring 기반 모작이 정확. 공식 문서 텍스트에선 직접 재확인 못함. `[추정 — 단일 출처, 구체적이라 신빙성 있음, 실물 확인 필요]`
- **클릭 하이라이트**: 공식 Cursor 가이드엔 **언급 없음**(부재 확인). 제3자 1건이 "soft pulse" 언급하나 신뢰도 낮음. 참고로 현행 Recap은 리플 하이라이트를 이미 구현 → 이건 Screen Studio 대비 초과 가능성. `[관찰됨 — 공식 부재]`

## 녹화 시작 플로우 (사용자 실제 앱 스크린샷 — `user-*.png`)

- **온보딩/권한 화면** (`user-01`): 거의 순검정(~`#0A0A0B`) 큰 라운드 창, 중앙 정렬. 보라 링 로고 → "Welcome to Screen Studio!" 흰 볼드 → 회색 서브텍스트. 권한 항목이 2열(좌: 권한명+회색 설명 / 우: 액션 버튼): "Allow Screen Recording"(다크 배경+보라 텍스트), "Accessibility access enabled"(초록 체크+다크그린 배경). Usage data 토글(off). 하단에 동의 체크박스 + "Accept and Continue"(권한 미완료 시 흐림/비활성). `[관찰됨 — 실물]`
- **녹화 컨트롤 바** (`user-02`): 알약형 다크 바, 큰 라운드. 좌측부터 X 닫기(원형 흰 버튼) → Display / Window / Area / Device 세그먼트(선택 항목만 밝게 하이라이트) → 구분선 → No camera / No microphone / No system audio → 설정 기어+드롭다운. `[관찰됨 — 실물]` (우리 범위: Device·camera·mic·system audio 제외 → Display/Window/Area + 설정만)
- **Window 선택 효과** (`user-03`): 대상 창만 밝게, 나머지 화면은 파랑빛 딤 오버레이. 창 중앙에 앱 아이콘 + 이름 + "1280 × 1415 Resize" 칩 + 보라 "Start recording" 버튼(+드롭다운 화살표). 하단 중앙 "Adjust recording area" 칩. `[관찰됨 — 실물]`

## 확보 불가로 확정 → 합리적 추정으로 구현

- **정밀 줌 이징(spring vs ease)**: export MP4를 사용자가 구독 없이 만들 수 없어 원본 프레임 측정 불가. → 공개 데모 관찰(약 0.5초, 부드러운 가감속, 오버슈트 미관찰, hold 중 미세 push-in)을 근거로 구현하고 실물과 나란히 비교하며 조정.
- **Animation 탭의 스프링 파라미터 UI(Tension/Friction/Mass 실재 여부)**: 앱이 해당 패널을 상위 레이어로 띄워 OS 스크린샷에 캡처되지 않음. → 확정 불가. 구현은 조정 가능한 파라미터(지속시간·이징/스프링 강도)를 두되 UI 노출 형태는 자유.

## 사용자 자료가 필요한 항목 (공개 자료로 확정 불가)

**정적(스크린샷으로 해결):**
1. 프리뷰 캔버스에 배경 프리셋 적용된 최종 룩 — 그라디언트 발색, 그림자 세기, 코너 라운드 px, 패딩 비율
2. 사이드바 전체 탭 목록·순서·명칭(Body/Captions 등 실제 내용)
3. New Recording 모달의 실제 레이아웃(토글 배치·아이콘·색)
4. Export 다이얼로그 실제 드롭다운 값·품질 단계·버튼 배치
5. 정확한 hex 색값(실제 스크린샷에서 스포이드 추출)
6. 라이트 모드 지원 여부

**모션(짧은 화면 녹화로 해결) — 가장 중요:**
7. 줌 인/아웃 이징이 ease-in-out인지 spring(오버슈트/바운스 유무)인지
8. 줌 전환 지속시간(ms)
9. 줌 배율(클릭당 몇 %)
10. 패닝 이동의 가속/감속 특성
11. 모션 블러 기본값 강도
12. 커서 스무딩 체감 강도(지연·오버슈트)
13. 클릭 시 링/리플 이펙트 실재 여부

## 공식 데모 영상 프레임 분석 (직접 수집)

screen.studio 랜딩의 데모 MP4를 직접 받아 ffmpeg로 프레임 추출해 관찰. 모두 1280×720 또는 720×720, **60fps**. 원본·몽타주는 `screen-studio-samples/`.

- **배경**: Raycast 스타일 고채도 멀티컬러 그라데이션(오렌지-핑크-마젠타-그린-퍼플) + iOS/macOS 월페이퍼 계열. 창은 라운드 코너 + 부드러운 그림자 + 패딩. `[관찰됨 — hero-demo]`
- **모션 블러 실재·강함**: 스크롤·줌 전환 시 텍스트가 이동 방향으로 뚜렷하게 번지는 방향성 모션 블러가 기본값에서 확연히 보인다. 픽셀 모작 필수 요소. `[관찰됨 — motion-blur.mp4]`
- **줌 전환**: 약 0.5초 내외의 부드러운 가감속. 명백한 스프링 오버슈트/바운스는 이 샘플들에서 관찰되지 않음(ease-in-out 또는 매우 약한 스프링). 정밀 spring/ease 확정은 사용자 export 영상 필요. `[관찰됨(정성) — motion-blur.mp4]`
- **줌 유지 중 미세 push-in**: 줌인 완료 후 hold 구간에도 화면이 정지하지 않고 아주 천천히 계속 확대되는 drift가 보인다. 현행 Recap의 정적 hold와 다를 수 있는 지점. `[관찰됨 — hero-demo 0.5~1.3s]`
- **종횡비 프리셋**: Presets 드롭다운에 Landing page(16:9)·Social Media Square(1:1)·TikTok and Instagram(9:16). `[관찰됨 — presets.mp4]`
