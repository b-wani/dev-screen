# 에디터 화면 상세 설계 (초안 — HITL #58)

> 상태: **DRAFT**. 기능은 경량 편집 헌장 그대로, UI/UX만 상향. 본 문서는 확정본이 아니라
> 사람이 결정할 갈림길(문서 끝 "Open decisions")을 남긴 초안이다.
>
> 근거 모델: `src/shared/recipe.ts`(RenderRecipe) · `recipe.edit.ts`(경량 편집 변환) ·
> `export-preset.ts`(익스포트 프리셋). 토큰: `docs/design/hoppy-design-language.md` (C-dark).
> 토폴로지: #54 — 에디터는 독립 문서창(다중 가능), 전역 상태머신에 preview 상태 없음,
> 메뉴바 전용 앱.

---

## 0. 원칙 (변하지 않는 것)

- **편집 = 렌더 레시피 수정.** 원본 녹화(영상+이벤트 트랙)는 절대 건드리지 않는다.
- **경량 편집만.** 트리밍 · 줌 구간(이동/길이/배율/삭제) · 배경·패딩·라운딩·섀도 ·
  커서(크기/스무딩) · 오버레이 배지/키. **컷 편집·속도 조절·자막은 제외.**
- **UI/UX만 상향, 기능 추가 금지.** 아래 레이아웃은 기존 컴포넌트(PreviewView/Sidebar/
  Timeline/ExportPanel)가 이미 편집하는 필드를 Screen Studio급 크롬으로 재배치한 것이다.
  모델에 없는 필드(월페이퍼/이미지 배경, 블러)는 "settled"로 넣지 않고 open decision으로 남긴다.
- **마스코트는 크롬에 없음.** 에디터 툴바·우측 패널·타임라인 어디에도 Hoppy를 넣지 않는다.
  익스포트 완료 순간에만 허용.

## 1. 전체 레이아웃

독립 문서창 1개 = 녹화 1개. 창은 3영역 고정 그리드.

```
┌─────────────────────────────────────────────────────────────┐
│  타이틀 바:  ● 녹화이름   대상·길이 메타        [익스포트 ▸]   │  ← 상단 바
├──────────────────────────────────────────┬──────────────────┤
│                                          │                  │
│            프리뷰 캔버스                  │   우측 설정 패널   │
│      (그라디언트 배경 위에 뜬 프레임)      │   (단일 스크롤)   │
│                                          │                  │
├──────────────────────────────────────────┴──────────────────┤
│  ▶  00:04 / 00:41    [◀|] [|▶]      ← 재생 컨트롤            │
│  ┌── 클립 트랙 (필름스트립 + 트림 핸들) ──────────────────┐   │  ← 하단 타임라인
│  └── 줌 구간 트랙 (줌 세그먼트, 선택/이동/길이) ──────────┘   │
└─────────────────────────────────────────────────────────────┘
```

- **좌: 프리뷰**가 주인공. 어두운 캔버스(`--bg`) 위에 배경/패딩이 입혀진 프레임이 밝게 뜬다.
  캔버스는 `sampleComposition` 출력을 그대로 그리는 얇은 층(현행 유지).
- **우: 설정 패널** 고정폭(약 320px), 세로 스크롤 단일 컬럼. 탭 없음.
- **하단: 타임라인** 전폭. 재생 컨트롤 + 2트랙.
- **상단 바**: 좌측에 녹화 정체성(이름·대상·길이), 우측에 primary 액션.

### 1.1 상단 바 — #54 반영
- 독립 문서창이므로 **"← 목록" 인앱 내비게이션 제거.** 창 = 문서. 목록/새 녹화는 메뉴바/메뉴에서.
- 좌측: 녹화 이름 + 대상 배지(`전체 화면`/`창`) + 길이(mono, 트림 반영). 원본 길이는 보조.
- 우측: **`● 같은 대상 다시 녹화`**(rec 레드, small) + **`익스포트`**(grass primary) —
  두 버튼은 의미(rec vs go)가 다르므로 크기·위치로 구분(디자인 언어 버튼 위계).
- (Export 동선의 정확한 위치는 Open decision 3.)

## 2. 우측 설정 패널 — 섹션 구조

각 컨트롤을 RenderRecipe 필드에 1:1 매핑한다. **모델에 있는 것만 settled.**

### ① 배경  → `recipe.background`
| 컨트롤 | 레시피 필드 | 상태 |
|---|---|---|
| 그라디언트 프리셋 스와치(슬레이트/그래파이트/인디고/틸/플럼) | `background.type='gradient'` + `background.gradient` (from `GRADIENT_PRESETS`) | **settled** |
| 단색 스와치 + 컬러 피커 | `background.type='color'` + `background.color` | **settled** |
| 월페이퍼 / 이미지 배경 | *(모델에 없음 — `BackgroundKind='color'|'gradient'`)* | **OPEN #2** |

### ② 블러
| 컨트롤 | 레시피 필드 | 상태 |
|---|---|---|
| 배경 블러 세기 | *(모델에 없음 — `BackgroundStyle`에 blur 필드 없음)* | **OPEN #2** |

### ③ 프레임 (패딩·라운딩·섀도) → `recipe.background`
| 컨트롤 | 레시피 필드 | 상태 |
|---|---|---|
| 패딩 슬라이더 (0–40%) | `background.padding` [0, 0.4] | **settled** |
| 라운딩 슬라이더 (0–32px) | `background.cornerRadius` | **settled** |
| 드롭 섀도 토글 | `background.shadow` (0 ↔ `SHADOW_ON` 0.45) | **settled** |

### ④ 커서 → `recipe.cursor`  *(티켓 패널 목록엔 없으나 기존 레시피 필드)*
| 컨트롤 | 레시피 필드 | 상태 |
|---|---|---|
| 크기 세그먼트 (1x/1.5x/2x) | `cursor.size` (`CURSOR_DEFAULTS.sizes`) | **settled**, 단 위치는 OPEN #4 |
| 스무딩 세그먼트 (끔/약/강) | `cursor.smoothingMs` (`CURSOR_DEFAULTS.smoothingLevels`) | **settled**, 단 위치는 OPEN #4 |

### ⑤ 오버레이 배지 · 키 입력 → `recipe.badge` · `recipe.keystrokes`
| 컨트롤 | 레시피 필드 | 상태 |
|---|---|---|
| 뷰포트 크기 배지 토글 | `badge.visible` (라벨은 `viewport`에서 유도, 예 `1440×900`) | **settled** |
| 맥락(브랜치/커밋) 자유 입력 | `badge.contextLabel` | **settled** |
| 키 입력 오버레이 토글 | `keystrokes.overlayVisible` | **settled** |

### 컨텍스트 모드 (줌 구간 선택 시)
타임라인에서 줌 구간을 고르면 패널이 그 구간 편집으로 바뀐다(현행 동작 유지):
- 배율 세그먼트 1.5x/2.0x/2.5x → `setZoomSegmentScale` (`ZOOM_DEFAULTS.scales`)
- `줌 구간 삭제`(danger 레드) → `deleteZoomSegment`
- 빈 곳 클릭/Esc로 기본 패널 복귀.

> UI/UX 상향 포인트: 섹션 제목은 SF Pro Rounded, 값 표기(%, px, x, 배지 라벨)는 SF Mono +
> tabular-nums. 스와치/토글/세그먼트는 pill. 카드 면은 `--surface`, 입력은 `--surface-2`,
> hover는 `--surface-3`, 테두리는 초록 알파 한 겹.

## 3. 타임라인 구조

전폭. **재생 컨트롤 + 2트랙**(현행 단일 트랙을 클립/줌 두 레인으로 분리 = UI 상향).

### 3.1 재생 컨트롤
- **재생/일시정지** 토글(▶/⏸). Space 단축키(현행).
- **프레임 스텝** ◀| / |▶ (1/60s). ←/→ 단축키(현행).
- **타임코드** `현재 / 길이` — mono, tabular-nums. 트림 시작 기준 상대 시각 표기(현행).
- (스크럽은 트랙 클릭/플레이헤드 드래그. 루프는 트림 창 내 자동 반복 — 현행 유지.)

### 3.2 클립 트랙 (상단 레인)
- 원본 영상의 **필름스트립**(썸네일 스트립) — 시간축 오리엔테이션.
- **트림 핸들** 양끝(앰버 `--warn`). 끌어 최종 영상 범위 결정 → `trimRecipe`.
- 트림 밖(앞/뒤) 구간은 어둡게 딤 처리(최종 영상 제외 표시) — 현행 `.tl-trimmed`.
- **재생 헤드** 세로선이 양 트랙을 관통.

### 3.3 줌 구간 트랙 (하단 레인)
- 자동 유도된 줌 구간을 **뉴트럴 회록색(`--zoom` #44513d)** 블록으로. grass와 충돌 회피.
- 블록 본체 드래그 = 이동(`moveZoomSegment`), 양 가장자리 = 길이 조절(`resizeZoomSegment`).
- 선택된 구간은 grass 아웃라인(활성/선택 신호). 라벨은 길이(초, mono).
- 배율·삭제는 우측 컨텍스트 패널에서(2절).

> 두 트랙을 세로로 쌓아 클릭 대상(클립 vs 줌)이 겹치지 않게 한다. 현행은 한 트랙에
> 트림·줌이 겹쳐 있어 정밀 조작이 어렵다 — 분리가 이번 UI 상향의 핵심.

## 4. Export 동선

익스포트 = 렌더 레시피를 원본에 적용하는 후처리(미리보기 확인 후 실행). 포맷 MP4/GIF,
프리셋은 v1 GitHub 하나(`GITHUB_PRESET`, UI에 선택·커스텀 미노출 — export-preset.ts 명시).

- 트리거 → 인코딩 진행률(%) → 완료(용량 표기 + `Finder에서 열기`/`경로 복사`, 초과 시 경고).
  (`ExportPanel`의 idle/encoding/done/error 4상태 그대로 재사용.)
- **완료 순간에만 마스코트 허용** — 성공 토스트/영역에 Hoppy 1회.
- 진입점 위치(상단 바 primary vs 우측 패널 섹션)는 **Open decision 3.**

## 5. Settled vs Open 요약

**Settled (모델·헌장·#54가 이미 규정 — UI만 상향):**
- 3영역 레이아웃(프리뷰+우측 패널+하단 타임라인).
- 독립 문서창, 인앱 목록 내비 제거(#54).
- 배경=그라디언트 프리셋+단색, 패딩/라운딩/섀도, 커서 크기/스무딩, 배지/맥락/키 오버레이.
- 타임라인 2트랙 분리(클립+줌), 트림 핸들(앰버), 줌 세그먼트(회록색), 재생 컨트롤.
- 익스포트 4상태 흐름, v1 프리셋은 GitHub 하나.
- 토큰·타이포·마스코트 규칙(디자인 언어).

**Open (사람이 결정) — 아래 별도.**

## 6. Open decisions (HITL)

### D1. 프리셋(렌더 레시피 저장) 도입 여부  *(티켓 필수 항목)*
- A. **v1 도입 안 함** *(권장)* — 레시피는 이미 폴더별 자동 저장·복원(`saveRecipe`). "스타일
  프리셋 저장/불러오기"는 신규 기능이라 "기능 추가 금지" 헌장과 충돌. 창별 독립 상태(#54)와도
  결이 맞음.
- B. 경량 "스타일 프리셋"만 — 배경/패딩/커서 스타일 묶음 저장(줌·트림 제외). UI 상향 vs
  기능 추가 경계가 모호.
- C. 익스포트 프리셋 UI 노출 — export-preset.ts의 GitHub 프리셋을 드러내고 향후 프리셋 추가
  대비. 현재 값 하나뿐이라 실익 적음(별개 개념).

### D2. 배경 "월페이퍼/이미지" + "블러" 추가 여부  *(티켓 문구 ↔ 모델 충돌)*
티켓은 배경에 월페이퍼/이미지와 블러를 나열하지만, 확정 모델은 `BackgroundKind='color'|'gradient'`
뿐이고(주석: "이미지/월페이퍼는 범위 밖") `BackgroundStyle`에 blur 필드가 없다.
- A. **모델 그대로(그라디언트+단색, 블러 없음)** *(권장)* — "기능 추가 금지" 준수, 모델
  불변. 티켓 문구는 위시리스트로 간주하고 v2로.
- B. 이미지/월페이퍼 배경 추가 — `BackgroundStyle`에 `image` 종류 + 소스/맞춤 필드 확장(모델
  변경 = 기능 추가, 익스포트 렌더러·영속화 파급).
- C. 블러만 추가 — `BackgroundStyle`에 `backgroundBlur` 필드 추가(compose·샘플링 변경).

### D3. Export 진입점 위치 / 동선
- A. **상단 바 우측 grass primary `익스포트` 버튼** *(권장)* — 디자인 언어가 "그린=전진, 익스포트"를
  primary로 규정. 포맷(MP4/GIF)·용량은 버튼 옆 팝오버/하위 영역에서. 화면에서 가장 밀 액션이 명확.
- B. 현행 유지 — 우측 패널 하단 `④ 익스포트` 섹션. 상향폭 작지만 파급 최소.

### D4. 커서 컨트롤을 우측 패널에 유지할지  *(티켓 패널 목록 누락)*
티켓 우측 패널 목록은 배경·블러·패딩·오버레이 배지만 열거하고 커서를 빠뜨렸으나, `cursor.size`·
`cursor.smoothingMs`는 기존 레시피 필드다.
- A. **유지(별도 "커서" 섹션)** *(권장)* — 제거하면 기존 편집 기능 상실(=기능 축소). 티켓 누락은
  단순 열거 누락으로 판단.
- B. 티켓대로 패널에서 제외 — 커서 편집 진입점 사라짐(기능 후퇴, 비권장).
