# 라이브러리 창 상세 설계 — draft spec (ticket #60, map #52)

> HITL 초안. 확정 아님. 아래 "열린 결정" 섹션은 사람이 골라야 한다.
> 디자인 토큰: Hoppy C-dark (`docs/design/hoppy-design-language.md`). 하드코딩 색 금지, `var(--…)` 사용.
> 데이터 모델 근거: `src/main/storage.ts` (RecordingManifest) + `src/shared/ipc.ts` (RecordingSummary).

---

## 0. 배경 · 토폴로지 (#54 확정, 재작성 대상 아님)

- 라이브러리는 **신규 독립 창**(썸네일 그리드). 에디터는 독립 문서창(다중).
- 최근 목록 로직이 `IdleView`에서 → 라이브러리 창으로 **이전**된다. (IdleView는 캡처 대상 선택 + 녹화 시작만 남는 방향.)
- 앱은 **메뉴바(트레이) 상주** 앱. 트레이 '최근 녹화' 서브메뉴(최대 8개)는 **유지**.
- 항목 클릭 → **에디터 창을 연다**(기존 `openRecording(folder)` = `recording:open` 재사용, 미리보기/편집 상태 복원).

---

## 1. 실제 존재하는 항목 메타 (⚠ 데이터 모델 제약)

라이브러리 카드는 "있는 데이터"만 그린다. `RecordingSummary`가 현재 노출하는 값:

| 필드 | 타입 | 카드 표기 | 폰트 |
|---|---|---|---|
| `startedAt` | epoch ms | 날짜 (`formatDate` → "7. 9. 14:32") | SF Mono, tabular-nums |
| `durationMs` | ms | 길이 (`formatElapsed` → "01:23") | SF Mono, tabular-nums |
| `eventCount` | number | "이벤트 42개" | SF Mono (숫자) + SF Text |
| `thumbnailUrl?` | string | 썸네일 (없으면 🎬 폴백) | — |
| `folder` / `name` | string | 식별자 / 폴백 라벨(timestamp) | — |

**존재하지 않는 것 (카드에 못 그림 without 데이터 모델 변경):**
- **사용자 지정 이름/제목** — 없음. 라벨은 timestamp 폴더명뿐. → "이름 변경" 액션과 "제목 검색"은 manifest에 `title` 필드 신설을 요구한다. (열린 결정 3)
- **파일 용량(bytes)** — `RecordingSummary`에 없음(`ExportSaveResult`에만 있음).
- **해상도** — `manifest.target.width/height`에 존재하나 `RecordingSummary`로는 미노출. 원하면 summary 확장 필요.

→ **settled:** v1 카드 메타 = 날짜 · 길이 · 이벤트 수 (IdleView가 이미 쓰는 3종을 그대로 상속). 용량·해상도는 v1 카드에서 제외.

## 2. 창 레이아웃

### 2.1 창 크롬 (settled)
- 헤더 바: 좌측 타이틀 "라이브러리"(SF Pro Rounded), 우측 정렬·검색 컨트롤. 마스코트 **없음**(밀도 높은 크롬).
- 본문: 스크롤되는 썸네일 그리드. 카드 radius `--l`(16), 면 `--surface`, 미세 초록 알파 테두리.
- 창은 자체 창(BrowserWindow). 에디터 모드 리사이즈 로직(`window:editor-mode`)과 무관 — 독립 창.

### 2.2 그리드 카드 (settled)
IdleView `recent-item` 구조를 그리드 카드로 승격:
- 상단: 16:9 썸네일(`--surface-2` 바탕, `object-fit: cover`), 우하단에 길이 배지(pill, 반투명 배경 + mono).
- 하단 본문: 날짜(주 라벨, text-1) / "길이 · 이벤트 N개"(보조, text-2, mono 숫자).
- hover: 면 `--surface-3`로 상승 + 우상단 `⋯` 오버플로 버튼 페이드인.
- 클릭(카드 본체) = 열기(에디터 창).

### 2.3 정렬 · 검색 (settled + 열린 결정 1·3 연동)
- **정렬:** 기본 = 최신순(`startedAt` desc; storage가 이미 이렇게 정렬해 반환). 셀렉트로 최신순/오래된순/길이순 제공. (이벤트 수 정렬은 저가치 — 제외 권장.)
- **검색:** 있는 텍스트가 날짜뿐이라 v1 검색의 의미가 약함. → 검색 UI의 운명은 "이름 변경/제목" 결정(열린 결정 3)에 종속. 제목 없이 v1을 내면 검색 입력은 **날짜 필터**로 축소하거나 **숨김**.

## 3. 항목 액션

카드 hover 시 `⋯` 오버플로 메뉴 + 우클릭 컨텍스트 메뉴(동일 항목):

| 액션 | v1 상태 | 구현 근거 |
|---|---|---|
| **열기 (에디터 창)** | settled | 기존 `recording:open` 재사용. 카드 클릭 = 이 동작. |
| **파일 위치 열기 (Finder)** | settled | `shell.showItemInFolder` — `export:reveal`이 쓰는 것과 동일 패턴. 신규 IPC(폴더 경로) 1개. |
| **삭제** | settled (방식은 열린 결정 4) | 신규 storage 함수 + IPC. 그리드에서 즉시 제거. |
| **이름 변경** | ⚠ 열린 결정 3 | manifest `title` 필드 신설 필요 — 현재 데이터 모델엔 없음. |

파괴적 액션(삭제)만 레드; 나머지 뉴트럴. (레드 성역 원칙 준수.)

## 4. 소환 경로 (열린 결정 2)

| 경로 | 현황 | 제안 |
|---|---|---|
| 트레이 컨텍스트 메뉴 | 현재 '런처 열기' + '최근 녹화' 서브메뉴 | '라이브러리 열기' 항목 **추가**(settled). |
| 캡처 툴바 / 런처 창 버튼 | 없음 | 노출 여부 = 열린 결정 2. |

- **settled:** 트레이 메뉴에 '라이브러리 열기'를 넣는다(전체 브라우즈의 정식 진입점).
- **open:** 런처(IdleView) 창에도 라이브러리 버튼을 둘지 (열린 결정 2).

## 5. 트레이 '최근 녹화'(최대 8) vs 라이브러리 창 — 역할 분담 (settled)

| | 트레이 '최근 녹화' 서브메뉴 | 라이브러리 창 |
|---|---|---|
| 목적 | **빠른 재진입** — 최근 8개 바로 열기 | **전체 브라우즈·관리** |
| 범위 | 최신 8개 | 전부 |
| 표현 | 텍스트 라벨(timestamp)만 | 썸네일 그리드 + 메타 |
| 액션 | 열기 1종 | 열기 · 삭제 · 파일 위치 · (이름변경?) |
| 정렬/검색 | 없음(고정 최신순) | 있음 |
| 창 | 안 뜸(메뉴에서 바로 에디터) | 전용 창 |

→ 최근 서브메뉴는 "지난 것 하나 다시" 초고속 경로, 라이브러리는 "다 훑고 관리". 중복이 아니라 계층. 둘 다 `listRecordings`를 공유(트레이는 `.slice(0,8)`).

## 6. 빈 상태 (첫 사용) (settled)

- 마스코트 Hoppy **허용**(여백·브랜드 표면). 그리드가 비면 중앙 정렬 히어로:
  - 마스코트 일러스트(자산 확정 전 플레이스홀더).
  - 헤딩(SF Pro Rounded): "아직 담긴 녹화가 없어요"
  - 보조 카피(SF Text): 어디서든 `⌥⌘R`로 첫 녹화를 시작하라는 안내 — IdleView 기존 카피 톤 상속.
- 마스코트는 **빈 그리드에만**. 항목이 하나라도 생기면 그리드 크롬으로 전환(마스코트 사라짐).

## 열린 결정 (사람이 결정) — RETURN 참조

1. 그리드 vs 리스트: v1에 리스트 뷰 토글을 넣나, 그리드 전용인가.
2. 캡처 툴바/런처 창에도 라이브러리 진입 버튼을 노출하나.
3. '이름 변경' + 제목 기반 검색을 v1에 넣나(= manifest `title` 필드 신설).
4. 삭제 방식: 휴지통 이동(복구 가능) vs 영구 삭제, 확인 다이얼로그 유무.

## Settled 요약
- 카드 메타 3종(날짜·길이·이벤트 수), 기존 데이터만.
- 최신순 기본 정렬(storage가 이미 그렇게 반환).
- 트레이에 '라이브러리 열기' 추가, '최근 녹화' 서브메뉴(최대 8)와 계층적 역할 분담.
- 항목 클릭 = 에디터 창 열기(`recording:open` 재사용).
- 파일 위치 열기 = `shell.showItemInFolder`.
- 빈 상태 = 마스코트 히어로.
